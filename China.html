    分布式云文件管理系统

    点击解锁
    • 实现了http协议(简洁版本):
      • 客户端向服务器发送请求消息，服务器在TCP协议之上回复响应消息。
      • 实现了HTTP持久连接(HTTP persistent connection): 客户端可以重用到给定服务器的TCP连接。
      • 提供安全控制(safe control)，不允许客户端访问文档根目录以外的内存。
    • 创建了一个名为SurfStore的容错云文件存储服务(fault tolerant cloud-based file storage service) （客户端和服务器使用gRPC通信）：
      • SurfStore 服务由两个部分构成：
        • BlockStore: 存储这些块(blocks)，并在给定标识符时，检索并返回相应的块。
        • MetaStore: 管理文件的元数据(meta data)和文件名到块的映射（通过SHA-256进行散列编排(marshall)）。
      • 客户端的文件数据在本地数据库中以版本(version)的进行存储。 运行客户端时，会发生同步操作(sync)，并将新文件添加到基本目录后上传到云端， 其他客户端同步到云端的文件将被下载到基本目录，解决任何有冲突的文件。
      • 使用一致性哈希算法(Consistent Hashing Ring)在不同的BlockStore中存储和管理块。
      • 通过RAFT协议确保MetaStore具有容错性(falut tolerant)， 并且在少数服务器故障(minority of server failure)时，始终保持一致性(consistent)。 (Raft 模拟器)

    带缓冲区管理的B+树

    点击解锁
    • 在I/O层之上构建一个缓冲池(buffer pool), 该缓冲池包含页帧(page frame)和一个哈希表，该哈希表将帧号映射到页号。
    • 一个页帧具有PinCnt(页面是否正在被使用), 脏位(页面是否被修改), 参考位(用于LRU时钟算法)。
    • 实现缓冲替换策略(buffer replacement policy)和LRU时钟算法(LRU clock algorithm)。
    • 在缓冲池层之上构建一个B+树, 支持INSERT/DELETE操作。
    • 通过批量加载(bulk loading)初始化B+树，并在数量超过"fanout"时分割页面。
    • B+树支持插入int、double和char*变量。

    操作系统实现

    点击解锁
    • 实现了操作系统线程结构：如 Alarm() 函数用来call 定时器中断; Join() 函数用来 在子线程完成前另父线程保持睡眠；实现了semaphores用来保证原子性(atomicity)。
    • 为每个用户进程创建pageTable数据结构, 该结构将进程的虚拟地址(virtual address)映射到 物理地址(physical address)。
    • 实现文件系统system call: create, open, read, write, close, unlink, join, exit and exec.
    • 实现 按需调页(demand paging), 分页替换(page replacement) 用以释放物理页面框架(physical page frame)以处理页面错误(page falut)。
